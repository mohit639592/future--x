Redundant FD & Extraneous Attributes

Redundant Functional Dependency in DBMS

A Redundant Functional Dependency is a dependency that can be removed from the set of functional dependencies without affecting the overall closure of the set.

👉 If removing a functional dependency does NOT change the closure → it is redundant

In simple words, a redundant FD is unnecessary because it can be derived from other functional dependencies.

📊 Example

A → B
B → C
A → C

🔍 Analysis

  • A → B and B → C are given
  • Using transitivity: A → C can be derived
👉 A → C is Redundant

⚙️ How to Check Redundant Functional Dependency

  • Step 1: Remove one functional dependency
  • Step 2: Compute closure using remaining dependencies
  • Step 3: If removed dependency can still be derived → it is redundant
👉 Closure is used to verify redundancy

📊 Deep Example (GATE Level)

F = { A → B, B → C, A → C }

Check if A → C is redundant:

Remove A → C and compute closure of A:

A⁺ = {A, B, C}

Since we still get C using A → B and B → C, the dependency A → C is redundant.

Final Set: A → B, B → C

💣 Why Remove Redundant Functional Dependencies?

  • To simplify the set of dependencies
  • To achieve Minimal Cover
  • To remove unnecessary complexity
  • To improve database design

🤯 Common Confusion

Is redundant FD same as extraneous attribute?

❌ NO

👉 Extraneous Attribute → remove part of FD
👉 Redundant FD → remove whole dependency

⚡ Exam Trick

  • Remove one FD
  • Find closure
  • If still derivable → remove it
👉 Always simplify FD set before solving questions

📊 GATE Question: Redundant Functional Dependencies

Given:

F = { X → YZ, ZW → P, P → Z, W → XPQ, XYQ → YW, WQ → YZ }

🎯 Goal

Find redundant functional dependencies.

⚙️ Step 1: Check X → YZ

Remove it and compute X⁺:

X⁺ = {X}

Cannot derive Y or Z → NOT redundant ✅

⚙️ Step 2: Check ZW → P

Remove it and compute (ZW)⁺:

ZW⁺ = {Z, W}

Apply W → XPQ:

ZW⁺ = {Z, W, X, P, Q}

We can derive P → ZW → P is REDUNDANT ❌

⚙️ Step 3: Check P → Z

Remove it and compute P⁺:

P⁺ = {P}

Cannot derive Z → NOT redundant ✅

⚙️ Step 4: Check W → XPQ

No other FD gives XPQ → NOT redundant ✅

⚙️ Step 5: Check XYQ → YW

Remove it and compute (XYQ)⁺:

XYQ⁺ = {X, Y, Q}

Apply X → YZ:

XYQ⁺ = {X, Y, Q, Z}

Cannot derive W → NOT redundant ✅

⚙️ Step 6: Check WQ → YZ

Remove it and compute (WQ)⁺:

WQ⁺ = {W, Q}

Apply dependencies:

  • W → XPQ → gives X, P, Q
  • P → Z → gives Z
  • X → YZ → gives Y and Z
WQ⁺ = {W, Q, X, P, Z, Y}

We can derive both Y and Z → WQ → YZ is REDUNDANT ❌

✅ Final Answer

Redundant Functional Dependencies:

ZW → P
WQ → YZ

Final Reduced Set:

F = { X → YZ, P → Z, W → XPQ, XYQ → YW }
👉 Final Answer: Remove ZW → P and WQ → YZ

Extraneous Attribute in DBMS

An Extraneous Attribute is an attribute that is not required in a functional dependency. If removing an attribute does not change the meaning of the dependency, then it is called extraneous.

👉 If removing an attribute does NOT change the dependency → it is Extraneous (Redundant)

In simple words, extraneous attributes are unnecessary parts of a functional dependency. Removing them helps simplify the database design and is an important step in Minimal Cover and Normalization.

📊 Example

AB → C
A → C

🔍 Analysis

  • A alone determines C
  • B does not contribute
👉 B is Extraneous (redundant attribute)

Types of Extraneous Attributes

1. Extraneous Attribute in Left Side (LHS)

An attribute in the left-hand side is extraneous if it can be removed and the dependency still holds.

AB → C
A → C

Since A alone determines C, B is extraneous.

A → C

2. Extraneous Attribute in Right Side (RHS)

If multiple attributes exist on the right side, we check them individually.

A → BC

Split into:

A → B
A → C

Then check if any dependency is unnecessary.

⚙️ How to Find Extraneous Attribute

  • Step 1: Remove one attribute
  • Step 2: Find closure of remaining attributes
  • Step 3: If result is same → attribute is extraneous
👉 Attribute Closure is the key concept to detect extraneous attributes

📊 Deep Example (GATE Level)

AB → C
A → C
C → D

Check AB → C:

A⁺ = {A, C, D}

Since A alone determines C, B is extraneous.

A → C
C → D
👉 Reduced (Minimal) Functional Dependency Set

💣 Why Extraneous Attribute is Important?

  • Helps in finding Minimal Cover
  • Removes redundancy
  • Simplifies functional dependencies
  • Improves database design
  • Important for GATE and interviews

🤯 Common Confusion

Does removing an attribute always change dependency?

❌ NO

👉 If dependency still holds → attribute was unnecessary

⚡ Exam Trick

  • Try removing attribute quickly
  • Compute closure
  • If nothing changes → remove it
👉 Always simplify before solving questions

📊 GATE Question: Extraneous Attributes

Given Functional Dependencies:

F = { X → Z, XY → WP, XY → ZWQ, XZ → R }

🎯 Goal

Identify extraneous attributes and simplify the functional dependencies.

1. X → Z

Single attribute on LHS → nothing to remove ✅

⚙️ Step 1: Check XY → WP

Check if X is extraneous:

Remove X → check Y⁺

Y⁺ = {Y}

Y cannot determine W or P → X is NOT extraneous ❌

Check if Y is extraneous:

Remove Y → check X⁺

X⁺ = {X, Z, R}

X cannot determine W or P → Y is NOT extraneous ❌

👉 XY → WP is already minimal ✅

⚙️ Step 2: Check XY → ZWQ

Split RHS:

XY → Z
XY → W
XY → Q

Check XY → Z:

Since X → Z already exists

👉 Z is redundant on RHS ❌

Check XY → W:

From XY → WP, we already get W

👉 W is redundant ❌

Check XY → Q:

No other FD gives Q → keep it ✅

👉 Reduced: XY → Q

⚙️ Step 3: Check XZ → R

Check if Z is extraneous:

Remove Z → check X⁺

X⁺ = {X, Z, R}

Since X already gives R → Z is extraneous ❌

X → R

⚙️ Step 4: Final Simplification

Remaining dependencies:

X → Z
XY → WP
XY → Q
X → R

✅ Final Answer

Extraneous removed:

  • Z removed from XZ → R
  • Z and W removed from XY → ZWQ
👉 Final Minimal Set:

X → Z, XY → WP, XY → Q, X → R

Difference Between Extraneous Attribute and Redundant FD

Extraneous Attribute Redundant FD
Removes part of FD Removes whole FD
Used in canonical cover Used in simplification
Check attribute removal Check FD derivation
