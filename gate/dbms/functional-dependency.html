Functional Dependency (FD)

A Functional Dependency is a relationship between attributes in a table.

👉 If one attribute determines another → it is called Functional Dependency

Notation: A → B

👉 If we know A, we can uniquely determine B

📊 Example

STUD_ID NAME DEPT
101 Amit CSE
102 Ravi ECE

🔍 Analysis

  • STUD_ID → NAME
  • STUD_ID → DEPT
👉 STUD_ID uniquely determines NAME and DEPT

Types of Functional Dependency

1. Trivial FD

If B is subset of A → Trivial

Example: (A, B) → A

2. Non-Trivial FD

If B is not subset of A → Non-trivial

Example: A → B

3. Completely Non-Trivial FD

If A and B have no common attributes

💣 Important Concept

Functional Dependency shows relationship between attributes, not rows.

👉 It is used in Normalization and finding Candidate Keys

🤯 Common Confusion

Does A → B mean every value must be different?

❌ NO

👉 It means: same A value → same B value

🎯 GATE Important Points

  • A → B means A determines B
  • Used in normalization
  • Helps find candidate keys
