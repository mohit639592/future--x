Functional Dependency in DBMS

Functional Dependency (FD) is the foundation of normalization and database design.

What is Functional Dependency?

A → B means A determines B.

Types of FD

  • Trivial
  • Non-Trivial
  • Completely Non-Trivial

Armstrong’s Axioms

  • Reflexivity: If B ⊆ A, then A → B
  • Augmentation: If A → B then AC → BC
  • Transitivity: If A → B and B → C then A → C

Derived Rules

  • Union: A → B and A → C ⇒ A → BC
  • Decomposition: A → BC ⇒ A → B and A → C
  • Pseudotransitivity: A → B and BC → D ⇒ AC → D

Attribute Closure (VERY IMPORTANT)

Closure of A is all attributes determined by A.

Example:

R(A,B,C), FDs: A → B, B → C

A⁺ = {A, B, C}

Steps to Find Closure

  • Start with given attribute
  • Apply FDs repeatedly
  • Add new attributes
  • Repeat until no change

Candidate Key

A candidate key is the minimal set of attributes that determines all attributes.

Example:

R(A,B,C), FDs: A → B, B → C

A⁺ = ABC ⇒ A is candidate key

Partial Dependency

Occurs when attribute depends on part of composite key.

Transitive Dependency

A → B and B → C ⇒ A → C

GATE Tips

  • Practice closure problems
  • Focus on keys
  • Understand normalization link
