Important GATE Questions on Functional Dependencies

GATE Question 1

Question:

Consider the relation scheme R = {E, F, G, H, I, J, K, L, M, N} and the set of functional dependencies:

  • {E, F} → {G}
  • F → {I, J}
  • {E, H} → {K, L}
  • K → M
  • L → N

What is the candidate key for R (GATE-CS-2014)?

A. {E, F}

B. {E, F, H}

C. {E, F, H, K, L}

D. {E}

Solution

Step 1: Start with EFH

  • EF → G
  • F → I, J
  • EH → K, L
  • K → M
  • L → N

👉 EFH⁺ = {E, F, G, H, I, J, K, L, M, N}

👉 Covers all attributes → Super Key

Check minimality:

  • Remove E ❌ (cannot get K, L)
  • Remove F ❌ (cannot get I, J)
  • Remove H ❌ (cannot get K, L)

👉 Minimal → Candidate Key

✅ Correct Answer: B. {E, F, H}

GATE Question 2

Question:

In a schema with attributes A, B, C, D, E, the following functional dependencies are given:

  • A → B
  • A → C
  • CD → E
  • B → D
  • E → A

Which of the following functional dependencies is NOT implied by the above set (GATE IT 2005) ?

A. CD → AC

B. BD → CD

C. BC → CD

D. AC → BC

Solution (Step-by-Step)

We check each option using closure

Option A: CD → AC

  • CD → E
  • E → A
  • A → C

👉 CD⁺ = {C, D, E, A}

✔ A and C present → IMPLIED

Option B: BD → CD

  • B → D (already have D)

👉 BD⁺ = {B, D}

❌ Cannot get C → NOT IMPLIED

Option C: BC → CD

  • B → D

👉 BC⁺ = {B, C, D}

✔ Contains C and D → IMPLIED

Option D: AC → BC

  • A → B

👉 AC⁺ = {A, B, C}

✔ Contains B and C → IMPLIED

❌ NOT IMPLIED: B. BD → CD

GATE Question 3

Question:

Consider the relation scheme R(A, B, C, D) and the set of functional dependencies:

  • AB → C
  • C → D
  • D → B

Find the candidate keys for R.

A. {AB}

B. {AB, AC}

C. {AB, AC, AD}

D. {A}

Solution

Step 1: Check AB

  • AB → C
  • C → D
  • D → B

👉 AB⁺ = {A, B, C, D}

👉 Covers all attributes → Super Key

👉 Minimal → Candidate Key

Step 2: Check AC

  • C → D
  • D → B

👉 AC⁺ = {A, C, D, B}

👉 Covers all → Candidate Key

Step 3: Check AD

  • D → B
  • AB → C

👉 AD⁺ = {A, D, B, C}

👉 Covers all → Candidate Key

👉 Final Candidate Keys = {AB, AC, AD}

✅ Correct Answer: C. {AB, AC, AD}

GATE Question 4

Question:

Consider the relation scheme R(A, B, C, D, E) and the set of functional dependencies:

  • AB → C
  • BC → D

Find the candidate keys for R.

A. {A, B, E}

B. {A, B}

C. {B, C, E}

D. {A, B, C, E}

Solution

Step 1: Identify attributes not on RHS

  • FDs: AB → C, BC → D
  • RHS = {C, D}
  • Attributes NOT in RHS = {A, B, E}

👉 These must be part of every candidate key

Step 2: Check closure of ABE

  • AB → C
  • Now we have B, C → BC → D

👉 ABE⁺ = {A, B, E, C, D}

👉 Covers all attributes → Super Key

Step 3: Check minimality

  • Remove A ❌ (cannot get C)
  • Remove B ❌ (cannot apply AB → C)
  • Remove E ❌ (E cannot be derived from anywhere)

👉 Minimal → Candidate Key

✅ Correct Answer: A. {A, B, E}

GATE Question 5

Question:

Consider the relation scheme R(A, B, C, D, E, F) and the set of functional dependencies:

  • AB → C
  • C → D
  • CD → BE
  • DE → F
  • EF → A

Find all candidate keys for R.

A. {AB, BC, CE, EF}

B. {AB, BC, CE}

C. {AB}

D. {AB, CE}

Solution

Step 1: Check AB

  • AB → C
  • C → D
  • CD → BE
  • DE → F
  • EF → A

👉 AB⁺ = {A, B, C, D, E, F}

👉 Covers all → Candidate Key

Step 2: Check BC

  • C → D
  • CD → BE
  • DE → F
  • EF → A

👉 BC⁺ = {B, C, D, E, F, A}

👉 Covers all → Candidate Key

Step 3: Check CE

  • C → D
  • CD → BE
  • DE → F
  • EF → A

👉 CE⁺ = {C, E, D, B, F, A}

👉 Covers all → Candidate Key

Step 4: Check EF

  • EF → A
  • AB → C
  • C → D
  • CD → BE

👉 EF⁺ = {E, F, A, B, C, D}

👉 Covers all → Candidate Key

👉 Final Candidate Keys = {AB, BC, CE, EF}

✅ Correct Answer: A. {AB, BC, CE, EF}

GATE Question 6

Question:

Consider the relation scheme R(A, B, C, D, E, F) and the set of functional dependencies:

  • AB → C
  • C → DE
  • E → F
  • F → B

Find all candidate keys for R.

Solution

Step 1: Find attributes not on RHS

  • RHS = {C, D, E, F, B}
  • Attributes NOT in RHS = {A}

👉 A must be present in every candidate key

Step 2: Try with A + B

AB → C → DE → F → B

👉 AB⁺ = {A, B, C, D, E, F}

👉 Candidate Key

Step 3: Try with A + F

F → B → AB → C → DE

👉 AF⁺ = {A, F, B, C, D, E}

👉 Candidate Key

Step 4: Try with A + E

E → F → B → AB → C → DE

👉 AE⁺ = {A, E, F, B, C, D}

👉 Candidate Key

Step 5: Check A + C

C → DE → F → B

👉 AC⁺ = {A, C, D, E, F, B}

👉 Candidate Key

✅ Final Candidate Keys: {AB, AC, AE, AF}

GATE Question 7

Question:

Consider the relation scheme R(A, B, C, D, E) and the set of functional dependencies:

  • A → B
  • B → C
  • C → D
  • D → A

Find all candidate keys for R.

Solution

Step 1: Identify structure

A → B → C → D → A (cycle)

👉 A, B, C, D are all equivalent (each determines others)

Step 2: Check attribute E

👉 E is not present in any FD 👉 So E cannot be derived 👉 E must be in every candidate key

Step 3: Form candidate keys

  • AE⁺ = {A, B, C, D, E}
  • BE⁺ = {B, C, D, A, E}
  • CE⁺ = {C, D, A, B, E}
  • DE⁺ = {D, A, B, C, E}

👉 All cover full relation → Candidate Keys

✅ Final Candidate Keys: {AE, BE, CE, DE}

GATE Question 8

Question:

Given the set of functional dependencies:

  • XY → W
  • Y → Z
  • WZ → P
  • WP → QR
  • Q → X

Show that: XY → P

Solution

Step 1: Start with XY(Find closure of XY if it contains p then Yes otherwise NO)

  • XY → W ⇒ add W

👉 Now we have: {X, Y, W}

Step 2: Use Y → Z

  • Y → Z ⇒ add Z

👉 Now we have: {X, Y, W, Z}

Step 3: Use WZ → P

  • WZ → P ⇒ add P

👉 Now we have: {X, Y, W, Z, P}

👉 P is derived from XY

✅ Hence Proven: XY → P
